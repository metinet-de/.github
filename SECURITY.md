# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in any of our projects, please report it responsibly.

### How to Report

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, please contact us directly:

### What to Include

Please include the following information:

1. **Project/Repository Name**
2. **Description of the vulnerability**
3. **Steps to reproduce** (if applicable)
4. **Potential impact**
5. **Suggested fix** (if you have one)
6. **Your contact information** (for follow-up)

### Response Time

- **Initial Response:** Within 48 hours
- **Status Update:** Within 7 days

### Security Best Practices

When using our projects:

1. **Keep API keys secure** - Never commit keys to version control
2. **Update regularly** - Always use the latest stable version
3. **Review permissions** - Check what permissions extensions/apps request
4. **Monitor usage** - Keep an eye on API usage and logs
5. **Report issues** - If you see something, say something
